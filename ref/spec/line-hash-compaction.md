# Line-Hash History 存储实验 — 演化报告

## 概览

从 v0（全文存储）到当前 v2.7，经历了 3 个主要阶段、10 次增量优化，总空间从 **693.8 MB 降至 129.5 MB（−81.3%）**。

```
v0   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 693.8 MB
v1   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 529.8 MB   -23.6%
v2.0 ━━━━━━━━━━━━━━━━━━━━━━ 151 MB (est)        -78.2%
v2.4 ━━━━━━━━━━━━━━━━ 133.4 MB                   -80.8%
v2.7 ━━━━━━━━━━━━━━━ 129.5 MB                    -81.3%
     └ history ████████ 39.2 MB
     └ index   ██████████████████ 90.4 MB
```

---

## Phase 0: v0 — 全文存储（基线）

每条修订存完整 content：

```json
{"rev_id":"...","timestamp":"...","author":"butler","summary":"编辑: 刘备",
 "content":"# 刘备\n\n刘备（161年—223年）...全文...","size":1715,"diff":[...]}
```

**空间: 693.8 MB**（125,631 条修订，平均 5.5 KB/条）

---

## Phase 1: v1 — Snap + Unified Diff

用 unified diff 替代 content，snap 每 26 版一个。

| 组件 | 大小 |
|------|------|
| Snap（~20K × ~5KB） | ~100 MB |
| Patch（~105K × ~4KB） | ~420 MB |
| **合计** | **529.8 MB** |

```
693.8 → 529.8 MB   -164.0 MB  (-23.6%)
```

**问题**：Unified diff 包含全文所有行，patch 大小 ≈ 原文。read 时需要 apply 整个 diff 链。

---

## Phase 2: v2 Line-Hash — 概念验证

每行用 hash 标识，版本只存 hash 数组，行索引全局共享。

### v2.0 — 初始格式

```json
{"v":2,"t":"snap","id":"20260515-120000-a1b2c3",
 "ts":"2026-05-15T12:00:00+0000","au":"butler",
 "su":"编辑: 刘备","sz":1715,
 "ch":"sha256:abc123def456789...",
 "ln":["a1b2c3","d4e5f6","g7h8i9",...]}
```

估算空间（基于实测最终版 39.2 MB + 各优化项节省）：

| 组件 | 估算 |
|------|------|
| History（~125K entries） | ~58 MB |
| 行索引（hex 6 位，拼音 109 桶） | ~93 MB |
| **合计** | **~151 MB** |

vs v1 −72%，主要收益来自行级内容寻址消除了全文冗余。

---

### v2.1 — ts 整数化

ISO 字符串 `"2026-05-15T12:00:00+0000"`（25 字符）→ Unix 时间戳 `1778758959`（10 字符）。

**节省**: 每条 -15 字符 × 125K ≈ **−1.9 MB**（history）

---

### v2.2 — 删除 ch 字段

`"ch":"sha256:abc123def456..."` 约 71 字符。sha256 信息已编码在 id 引用的 content 中（去重时比较 id 而非 ch），ch 完全冗余。

**节省**: 每条 -71 字符 × 125K ≈ **−8.9 MB**（history）

---

### v2.3 — ln 数组→空格字符串

JSON 数组 `["a1b2c3","d4e5f6","g7h8i9"]` → 空格分隔 `"a1b2c3 d4e5f6 g7h8i9"`。

对于 N 行页面：
- 数组: `[` + N×`"xxxxxx"` + (N-1)×`,` + `]` = N×8 + (N-1) + 2 = **9N + 1** 字符
- 字符串: N×6 + (N-1)空格 = **7N - 1** 字符
- 节省: **2N+2** 字符/条

假设平均每页 ~100 行（含空行、frontmatter），125K entries：

**节省**: (2×100+2) × 125K ≈ **−3.2 MB**（history）

---

### v2.4 — id 改用 base62

`"20260515-120000-a1b2c3"`（28 字符）→ `"UENCcE"`（6 字符）。去掉时间戳前缀，用 base62 编码 sha256。

**节省**: 每条 -22 字符 × 125K ≈ **−2.75 MB**（history）

同时去重更可靠：相同内容必然相同 id，直接比较末条 id。

---

### v2.5 — su 摘要哈希化

摘要文本 `"编辑: 刘备"`（~10-50 字符）→ hash 引用 `"99e33a"`（6 字符）。

行索引新增摘要文本的映射条目（索引从 92.9 MB → 94.2 MB，+1.3 MB）。

**节省**: 每条约 -30 字符 × 125K = −3.75 MB；索引 +1.3 MB

**净节省**: **−2.45 MB**（总空间）

---

### v2.6 — 行 hash hex→base62

hex 截断 `"a1b2c3"`（16^6 ≈ 16.7M 空间，0.91% 碰撞率）→ base62 截断 `"K7p2Nm"`（62^6 ≈ 56.8B 空间，0% 碰撞率）。

hash 长度不变（6 位），**空间无变化**。但可靠性显著提升——524,893 个唯一行中 **0 碰撞**，无需延长到 7 位。

---

### 格式优化小结（v2.0 → v2.6）

| 步骤 | 改动 | History 节省 | 索引影响 | 累计 |
|------|------|-------------|---------|------|
| v2.0 | 初始 line-hash | — | 93 MB | ~151 MB |
| v2.1 | ts 整数化 | −1.9 MB | — | ~149 MB |
| v2.2 | 删除 ch | −8.9 MB | — | ~140 MB |
| v2.3 | ln 字符串化 | −3.2 MB | — | ~137 MB |
| v2.4 | id base62 | −2.75 MB | — | ~134 MB |
| v2.5 | su 哈希化 | −3.75 MB | +1.3 MB | ~133 MB |
| v2.6 | 行 hash base62 | — | — | ~133 MB |

这些优化主要压缩 **history JSONL 文件**（从 ~58 MB 降至 **39.2 MB**，−32%），行索引大小基本不变（含新增的摘要索引）。

## Phase 3: 行索引分桶优化 — 拼音→Hash

### v2.7 — Hash 分桶

**问题**：拼音分桶（109 桶）导致不同页面中的相同行内容跨桶重复存储，且分布极不均匀（`di/` 46K 行 vs `a0/` 17 行）。

**方案**：全站共享行索引，按 hash 首两字符分 992 桶：

```python
def hash_bucket(h: str) -> str:
    first = h[0]                                   # 62 种
    second = format(BASE62_ALPHABET.index(h[1]) % 16, 'x')  # 16 种
    return first + second                          # 62×16=992 桶
```

| 指标 | 拼音分桶 | Hash 分桶 | 变化 |
|------|---------|----------|------|
| 桶数 | 109 | **976** | +8× |
| 唯一行数 | 591,083 | **524,893** | **−11.2%** |
| 索引大小 | 94.2 MB | **90.4 MB** | **−3.8 MB** |
| 桶均匀度 | 17~46K 行/桶 | **~540 行/桶** | 均匀 |
| 平均桶大小 | ~800 KB | **~95 KB** | 利于前端缓存 |

**节省**: 索引 **−3.8 MB**（−4%）

全局去重节省的来源——相同行内容在不同拼音桶页面中重复出现（如空行、标点行、引文编号行等），hash 分桶后只存一份。

## 当前架构

```
编辑页面 → record_revision.py
               │
               ├─ 写入 history/<bucket>/<page>.jsonl
               │     (v2 snap/delta, base62 id)
               │
               ├─ 更新 line_index/<hash_bucket>.json
               │     (新行自动注册)
               │
               └─ 追加 recent.lite/diff.jsonl
                     (滚动窗口 1000 条)

浏览页面 → renderer.js
               │
               ├─ 读 history/<bucket>/<page>.jsonl
               ├─ 懒加载 line_index/<hash_bucket>.json
               │     (按 hash 自动定位桶，缓存复用)
               └─ 重建全文 → 渲染
```

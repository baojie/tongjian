# SKILL W6: 原文页 Wikification 标注规范

## 一、核心原则

1. **链接专有名词，不链接普通名词。** 只对特定命名实体加 `[[...]]`，普通器物、动作、性状不加。
2. **拆到合理粒度。** 不将整个描述性短语作为单个链接，按语义成分拆解。
3. **有源可循。** 优先链接已有 wiki 页面的实体；暂无页面的也可加链接（broken-link 接受）。
4. **一段不重复。** 同一段落内同一实体只链接一次（首次出现）。
5. **不嵌套。** `[[...]]` 内不再包含 `[[...]]`。

## 二、链接对象（明确要链接）

### A1 — 人物
- 所有有名有姓的人物：`[[商鞅]]`、`[[司马光]]`、`[[项羽]]`、`[[刘邦]]`
- 称谓/身份：`[[秦孝公]]`、`[[汉武帝]]`、`[[赵武灵王]]`
- 绰号/别号：`[[卫鞅]]`（商鞅原名）、`[[亚父]]`（范增）

### A2 — 地点
- 国家/地区：`[[秦国]]`、`[[赵国]]`、`[[楚国]]`、`[[关中]]`
- 地理名称：`[[函谷关]]`、`[[鸿门]]`、`[[垓下]]`、`[[长安]]`
- 都城/城邑：`[[咸阳]]`、`[[洛阳]]`、`[[建业]]`
- 战略要地：`[[井陉]]`、`[[潼关]]`、`[[赤壁]]`

### A3 — 官职/爵位/制度（特定命名）
- 官职：`[[丞相]]`、`[[太尉]]`、`[[尚书令]]`、`[[节度使]]`
- 爵位：`[[列侯]]`、`[[彻侯]]`、`[[左庶长]]`
- 制度：`[[郡县制]]`、`[[均田制]]`、`[[府兵制]]`

### A4 — 文献/典籍
- 典籍书名：`[[史记]]`、`[[汉书]]`、`[[孙子兵法]]`
- 著名诏令：`[[求贤令]]`、`[[轮台罪己诏]]`

### A5 — 事件
- 重大历史事件：`[[商鞅变法]]`、`[[赤壁之战]]`、`[[玄武门之变]]`
- 重要战役：`[[长平之战]]`、`[[官渡之战]]`、`[[淝水之战]]`

### A6 — 概念/组织
- 概念：`[[连横]]`、`[[合纵]]`、`[[均输法]]`
- 组织/制度：`[[三省六部]]`、`[[九品中正制]]`
- 朝代：`[[西汉]]`、`[[东晋]]`、`[[北魏]]`

## 三、不链接（明确不链接）

| 类别 | 示例 | 原因 |
|------|------|------|
| 通用颜色 | 红的、黄的 | 普通形容词 |
| 通用材质 | 丝绸、铁器、布帛 | 普通名词 |
| 通用称谓 | 将军、大夫、士卒 | 普通名词（非特定人物） |
| 通用器物 | 弓箭、战车、旌旗 | 普通名词 |
| 动作/性状 | 行军、攻城、议政 | 动词/形容词 |
| 代词称谓 | 朕、吾、君、彼 | 代词 |
| 时间词 | 是年、翌日、顷之 | 功能词 |

## 四、拆解规则（关键）

描述性短语必须按语义成分拆解，禁止整串链接。

### 4.1 官职+人名拆解

```
左丞相萧何
├── 左丞相 ← 官职（链接）
└── 萧何 ← 人名（链接）

淮阴侯韩信
├── 淮阴侯 ← 爵位（链接）
└── 韩信 ← 人名（链接）
```

### 4.2 地名+事件拆解

```
鸿门宴
└── 鸿门宴 ← 整体可链（知名历史事件名）

赤壁之战
└── 赤壁之战 ← 整体可链（战役名）
```

### 4.3 事件/活动拆解

```
商君变法令
├── 商君 ← 人名
└── 变法令 ← 内容（可链[[变法]]或保留不链）

三家分晋
└── 三家分晋 ← 整体链入（标准事件名）
```

### 4.4 原则总结

- **已有独立词条的** → 直接链接该词条名
- **无独立词条但明显是特定命名的** → 可加链接（允许 broken-link）
- **描述性短语由多个独立概念组成** → 拆开分别链接
- **无法拆解的整体命名** → 整体链接（如战役名、事件名）

## 五、工作流程

### 每卷流程

```
步骤 1  读原文：cat wiki/public/pages/di/第NNN卷.md
步骤 2  标注：扫描原文，找出所有应链未链的实体
步骤 3  替换：用 edit_page.py 写入
步骤 4  确认变更：
     PAGE=$(python3 -c "from wiki.scripts.page_bucket import resolve_page_file; from pathlib import Path; print(resolve_page_file(Path('wiki/public/pages'), '第NNN卷'))")
     head -5 "$PAGE"
```

### 批处理策略

- **单卷处理**：每轮处理 1 卷，确保质量
- **批量上限**：每卷新增链接 ≤ 100 处（超量则拆为两轮）
- **已有链接**：不修改已有 `[[...]]`，只补充缺失
- **前后一致**：同一实体在同一卷内保持命名统一

### 特殊情况

- 对话中的人名 → 链接，如「[[商鞅]]」「[[孝公]]」
- 引用的典籍名 → 链接，如「《[[史记]]》」

## 六、质量检查

每卷完成后自查：

1. 是否有应链未链的实体？→ 补充
2. 是否有整串模糊链接应拆解？→ 拆解
3. 是否有普通名词误链？→ 移除
4. 是否有一段内重复链接？→ 保留首次

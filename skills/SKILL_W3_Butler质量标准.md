# SKILL W3: 质量标准

## 一、质量等级定义

| 等级 | 中文 | 判定条件 |
|------|------|---------|
| `stub` | 存根 | 散文 < 100 字，或无 h2 节且全文 < 200 字 |
| `basic` | 基础 | 散文 < 400 字，或（h2 节 < 2 且 PN < 2） |
| `standard` | 标准 | h2 节 ≥ 2 + PN ≥ 2 + 散文 ≥ 400 字 + wikilink ≥ 3 |
| `featured` | 精品 | h2 节 ≥ 3 + PN ≥ 5 + 散文 ≥ 600 字 + wikilink ≥ 5 + 有他人评价或臣光曰 |
| `premium` | 旗舰 | featured 全部达标，且**旗舰评分 ≥ 5 分**（见下方评分矩阵） |

卷页面（type=章节）不参与评级。

---

## 二、旗舰评分矩阵（featured → premium）

参考三体 Wiki 精品页识别方法：每项 1 分，**≥ 5 分即达 premium**。

| # | 维度 | 达标条件 |
|---|------|---------|
| P1 | **PN 密度** | PN 引注 ≥ 10 条 |
| P2 | **多角度评价** | 他人评价（君主/同僚/对手/史家）原文引用 ≥ 3 人次 |
| P3 | **臣光曰** | 收录司马光史论原文（含 PN），完整引用 ≥ 1 段 |
| P4 | **跨卷视角** | 引文来源跨 ≥ 3 个不同卷号（非集中于 1-2 卷） |
| P5 | **深度散文** | 分析性段落（性格/制度背景/历史影响等）≥ 300 字 |
| P6 | **结构完整** | 有本类型必要节（见下方类型必要节表） |
| P7 | **知识图联通** | wikilink ≥ 8，且包含相关事件/战役页双向链接 |

**快速排除**：type=章节、type=名句、type=综述 — 不参与旗舰评分。

---

## 三、升级路径

```
stub    → basic    ：写 300 字散文，加 2 节，找 1-2 条 PN
basic   → standard ：扩散文到 400 字，h2 节 ≥ 2，PN ≥ 2，wikilink ≥ 3
standard→ featured ：h2 节 ≥ 3，PN ≥ 5，散文 ≥ 600 字，补他人评价或臣光曰
featured→ premium  ：旗舰评分矩阵补至 5 分（优先补 P2/P3/P5 缺口）
```

## 四、类型必要节（P6 达标条件）

| type | 必须有的节 |
|------|-----------|
| 人物 | 主要事迹（或生平）+ 历史评价（或后世影响） |
| 事件 | 经过 + 历史影响 |
| 战役 | 经过 + 结果与影响 |
| 国家 | 兴亡历程 + 历史地位 |
| 概念 | 含义/定义 + 历史沿革 |
| 地点 | 地理位置 + 相关历史事件 |

## 五、资治通鉴内容质量指标

### PN 引注铁律（所有页面类型均适用）

**每条事实陈述后必须附 PN**，包括：
- 人物的行为、言语、命运描述 → `（NNN-PPP）`
- 解释"为什么"的原因陈述 → `（NNN-PPP）`
- 地点的位置、特征、场景描述 → `（NNN-PPP）`
- 事件的经过与结果 → `（NNN-PPP）`

无原文依据的陈述一律不写，写"（待考证）"。

**Blockquote 之间空行铁律**：每条 `>（NNN-PPP）` 结尾的引文块之后，若紧接另一条 `>` 引文块（中间无空行），必须在两块之间插入空行。多条独立引用块不得挤在一起。示例：

```markdown
<!-- 错误 -->
> 原文一（010-011）
> 原文二（010-013）

<!-- 正确 -->
> 原文一（010-011）

> 原文二（010-013）
```

**人物页** standard 级至少需要：
- [ ] 基本信息节（身份/朝代/官职）
- [ ] 主要事迹节（≥ 2 条，每条含 PN）
- [ ] PN 引文 ≥ 3 条（格式：`（NNN-PPP）`）
- [ ] 至少 3 个 [[wikilink]]
- [ ] description 字段非空

**地点页** standard 级至少需要：
- [ ] 位置描述（含 PN）
- [ ] 主要相关人物/事件（含 PN）
- [ ] 重要历史场景 ≥ 1 条（含 PN）
- [ ] PN 引文 ≥ 2 条

**概念页** standard 级至少需要：
- [ ] 含义说明（≥ 100 字，关键定义处含 PN）
- [ ] 与通鉴核心历史事件的关联（含 PN）
- [ ] PN 引文 ≥ 2 条

## 六、与 Wikipedia 质量差距分析

### 对比方法

将主页精品页与 Wikipedia 对应页面逐项对比，识别内容缺口。

### 对比结果

| 维度 | 本 Wiki 现状 | Wikipedia 水平 | 差距 |
|------|-------------|---------------|------|
| 原文引注（PN） | 12-19 条/页，强 | 5-13 条/页 | ✅ 领先 |
| 事迹记叙 | 详细，大事表+散文 | 分散在各节 | ✅ 更清晰 |
| 信息框（Infobox） | ❌ 全无 | ✅ 关键事实一览 | 🔴 缺失 |
| 人物性格分析 | 蜻蜓点水（2-3句） | 独立整节，多角度 | 🔴 需要深化 |
| 历史评价 | 部分有 | 各家史学评价 | 🟡 部分缺失 |
| 相关史料 | ❌ 全无 | 引用史学论著 | 🔴 缺失 |
| 各家学说/争议 | ❌ 全无 | 不同历史解读 | 🔴 缺失 |
| 朝代/制度背景 | 部分有 | 独立整节 | 🟡 部分缺失 |
| 外部考据引用 | ❌ 全无 | 引用史学著作 | 🔴 缺失 |
| 司马光评（臣光曰） | 部分有 | 列入独立节 | 🟡 部分缺失 |

### 关键发现

1. **Infobox 是最大的缺口**。Wikipedia 所有完善的人物页都有信息框，一眼可见人物的朝代、官职、生卒、成就等关键信息。
2. **历史评价是第二大缺口**。Wikipedia 引用[[司马光]]、[[班固]]等各代史家评价，展现不同视角。本 Wiki 目前叙述偏重事实记录，对后世史学评价覆盖不足。
3. **臣光曰的收录是第三大缺口**。《资治通鉴》中司马光的史论（臣光曰）是本书的核心价值之一，应系统收录于相关词条。
4. **PN 引注是本 Wiki 的强项**。Wikipedia 记事常无原文佐证，本 Wiki 的 PN 体系确保每条叙述可回溯至原著段落。这点必须坚持并强化。

### 升级方向

从 featured → premium 需补齐以下维度：

```
基础层（必须）：
  □ 信息框（Infobox）—— 关键事实一目了然
  □ 事迹记叙 + PN 引注（本 Wiki 已有，保持）

进阶层（推荐）：
  □ 人物性格分析（≥200字，引用原文描写）
  □ 历史评价（各代史家观点）
  □ 臣光曰完整收录（人物/事件相关）

增值层（加分）：
  □ 相关史料引用
  □ 朝代制度背景
  □ 史学研究成果
```

## 七、W3 自评规则

每页完成后立即自评：

**accept 条件**：
- 内容有语料依据（corpus_search 可验证）
- Frontmatter 字段完整（id/type/label/description）
- PN 引文格式正确（`（NNN-PPP）`，非捏造）
- **每条事实陈述均附有 PN**，无裸陈述（无 PN 的事实一律写"（待考证）"）
- Wikilink 格式正确（`[[词条名]]`）
- **引文与页面话题直接相关**：blockquote 引用的原文必须与页面主题（人物/地点/事件）直接关联，禁止引用仅上下文提及主题、但引文本无关的段落
- 无明显事实错误

**fail 条件**：
- PN 来自猜测而非 corpus_search
- 内容与原文矛盾
- 覆盖了已有内容（追加原则违反）
- Frontmatter 缺失必填字段
- **blockquote 引文与页面主题不直接相关**（如项羽页面引用刘邦独自行军的段落）

**skip 条件**：
- 页面已存在且质量已达 featured 以上
- corpus_search 命中 0 条
- 任务超出当前卷目知识范围

## 八、引文相关性审计

### 背景

全库扫描发现，大量人物页（type=人物）引用了**与页面主题无直接关联**的原文 blockquote。这些引文的原文段落虽然出现在同卷同场景，但核心叙事对象并非页面主题人物，造成"借题发挥"式的引文质量问题。

### 审计范围与方法

| 维度 | 说明 |
|------|------|
| 筛选对象 | 所有 type=人物 的页面 |
| 初步筛查 | heuristic：检查 blockquote 中是否出现页面 label/aliases 的名称；若未出现且文中含其他主要人物名，标记为候选 |
| 优先级 P1 | 引文核心叙事对象是**另一人物** — **必须替换** |
| 优先级 P2 | 引文为场景/背景描写（仅上下文提及主题人物）— **视情况替换** |
| 优先级 P3 | 引文确与主题相关（主题人物在引文中扮演核心角色）— **保留** |

### 发现模式

最常见的三种不相关引文模式：

| 模式 | 示例 | 严重程度 |
|------|------|---------|
| A：引用其他人物的重要情节 | 商鞅页面引用张仪连横的段落 | P1 🔴 |
| B：引用场景背景描写 | 项羽页面引用汉军将领会议的段落 | P1 🔴 |
| C：引用次要情节 | 司马光页面引用宋神宗独自批奏的段落 | P1 🔴 |

### 修复策略

```
1. 识别：用 corpus_search 检查候选引文是否直接提及页面主题
2. 替换搜索：corpus_search "主题人物名+关键词" --vol N --max 5
3. 确认相关性：新引文必须满足——
   a. 主题人物在引文中是核心参与者/说话者
   b. 引文内容直接展现主题人物的行为、言语、性格或命运
4. 替换：通过 Edit 工具替换 blockquote（保留页面原有 ## 节结构）
5. 验证：PN 格式正确，surface form 与 corpus 一致
```

### 防止复发

- 新建页面时，blockquote 须来自 corpus_search 中直接提及该词条的段落
- enrich 操作补充引文时，优先选择该词条为句中主语的段落
- 自评（W3 accept 条件）中已加入"引文与页面话题直接相关"的检查项

# SKILL W9: 页面图式反思

> 内容质量的"进化"发生在这里——W5 改进 butler 规则，W9 改进页面结构本身。

---

## 一、触发时机

**每轮 butler 结束后**（即：完成精品页升级 + W6 检查后），执行一次 W9 反思。

与 W5 不同：
- W5 是系统性反思（每 29 轮触发），修订 W1-W4 规则
- W9 是内容性反思（每轮触发），修订页面模板/分类体系
- W9 **不阻塞**下一轮，不写 failures.jsonl，只写 schema_patterns/

---

## 二、反思流程（4 步）

### 步骤 1 · 收集本轮样本

```bash
# 查看本轮新建/升级的 featured 或 standard 页
python3 -c "
import json
data = json.load(open('wiki/public/pages.json'))
pages = data['pages']
premium = [k for k, v in pages.items() if v.get('quality') in ('featured', 'standard')]
print('\n'.join(sorted(premium)[:6]))
"
```

取本轮新建/升级的 **3 个 featured/standard 页** 作为"新样本"。

### 步骤 2 · 识别细分类型

对每个新样本，提取其 `tags` 字段，匹配**细分类型表**（见§四）。

- 若匹配到已有类型 → 找同类型历史页面（≥3 页）进行横向对照
- 若未匹配 → 判断是否应新建类型（≥2 页同构则新建）

**横向对照方法**：
```python
# 从 pages.json 中筛选同类型页面
import json
pages = json.load(open('wiki/public/pages.json'))['pages']
same_type = [p for p in pages.values() 
             if any(t in p.get('tags', []) for t in target_tags)]
```

### 步骤 3 · Schema 差距分析

对同类型的 3-6 个页面，比较：

| 检查项 | 理想状态 | 常见缺失 |
|---|---|---|
| **必有节名** | 细分类型的标准 sections 都在 | 缺"历史评价"节、缺"相关事件"节 |
| **frontmatter 字段** | 类型特有字段完整 | 将领缺"主要战役"、官员缺"任职朝代" |
| **引文比例** | 重要场景有原文 | 只有叙述无引文 |
| **链接密度** | 同类页面互相链接 | 同类人物各自孤立 |

### 步骤 4 · 输出

写 `wiki/logs/butler/schema_patterns/YYYY-MM-DD-R<N>.md`（每轮一个文件）：

```markdown
# 图式反思 2026-MM-DD R<N>

## 本轮样本
- [[页面1]]（细分类型：XXX）
- [[页面2]]（细分类型：YYY）
- [[页面3]]（细分类型：ZZZ）

## 同类横向对照

### 类型: 将领（本轮 +1，累计 12 页）

**当前 schema 覆盖情况**：
| 节 | 有 | 缺 |
|---|---|---|
| 基本信息 | 12/12 | - |
| 主要战役 | 10/12 | 2 缺 |
| 主要事迹 | 8/12 | 4 缺 |
| 原文引文 | 6/12 | 6 缺 |
| 历史评价 | 5/12 | 7 缺 |

**发现**：
- 所有将领页都应增加"主要战役"字段（基本信息中）
- 有独立战绩的将领（韩信/卫青/岳飞）应有"主要战役"节

**提案**：
1. 更新 `skills/templates/将领.md` — 增加"主要战役"行
2. 下轮遇到同类页面时，补充缺失的"原文引文"节

## 新类型识别

### 谏臣（本轮 +1：魏征，累计 3 页）

本轮识别出这是一个稳定细分类型，特征：
- cat=谏臣
- 都以直言进谏著称
- 多在名君时代出现，与帝王关系密切

**推荐 schema**（见 `skills/templates/谏臣.md`）：
- 必有节：基本信息 > 主要谏言 > 帝王关系
- 加分：与同一君主的其他臣子互链

→ 本轮已创建模板文件。

## 本轮无新发现的类型
- 帝王（汉高祖/唐太宗类）：已有成熟模板，无需更新
```

若某类型已有 ≥5 页且发现重大 schema 缺口 → **立即写/更新** `skills/templates/<类型>.md`。

## 三、模板文件规范

模板存于 `skills/templates/<细分类型>.md`，格式：

```markdown
---
subcategory: <细分类型名>
tags_pattern: [必须包含的 tag 之一]  
created: YYYY-MM-DD
updated: YYYY-MM-DD
sample_pages: [page1, page2, page3]
version: 1
---

# 模板：<细分类型名>

## 适用判断
满足以下任意 2 条即适用：
- cat 为 XXX
- 人物的核心身份是 YYY

## 必有节（缺一标红旗）

### 基本信息
（frontmatter + 结构化字段：朝代/官职/生卒）

### [细分类型特有节]
（本类型的核心内容）

### 人物关系表
必有行（按类型列出）：
| 君臣关系 | ... |
| 政治盟友/对手 | ... |
| 属下/上级 | ... |

### 原文引文（重要事迹必须有）

## 加分项
- 同类人物互链
- 司马光史论（臣光曰）收录
- 历史影响/后世评价
```

## 五、积累规则

1. **每轮写一个** `schema_patterns/YYYY-MM-DD-R<N>.md`（即使无新发现，也写"无新发现"）
2. **当某类型积累 ≥3 页**时，如果还没有模板文件，当轮必须创建
3. **当某类型积累 ≥5 页**时，如果模板文件超过 30 天未更新，当轮重新评估
4. **每 10 轮**回顾 `skills/templates/` 目录，更新类型索引表
5. 模板修订**必须**写进 `wiki/logs/butler/skill_changes.md`

# 高质量精品页建设方法论

## 快速开始

**适用场景**：对已有基础结构的 wiki 人物/事件页进行深度强化，达到 featured 精品质量。

**成功标准**（W8 建设完成的认证门槛）：
- 所有重要事迹均有 PN 引用（≥ 3 处）
- 多角度他人评价（≥ 3 人）——**以原文引用为主，非推理性陈述**
- 原文直接引用 ≥ 3 处引文块（不只是叙述）
- 相关事件页互联
- 跨卷互见：≥ 1 处来自非主角核心卷的他人评价

**⚠️ 核心原则：以事实收集为主，降低分析比重**

精品页的核心价值是**引文的广度与准确**，而非分析文字的长度。

| 优先 | 做 |
|---|---|
| ✅ 高优先 | 新增原文引文块（带PN） |
| ✅ 高优先 | 跨卷他人评语（原文+PN） |
| ✅ 高优先 | 补充事迹的具体原文证据 |
| ✅ 中优先 | 散文化叙述（串联引文的流畅过渡） |
| ⚠️ 低优先 | 分析段落（简短陈述即可，≤ 150字） |
| ❌ 避免 | 大段推理性分析、"这说明了……"式解读 |

**信息量评估**：同等篇幅下，一段原文引用比一段分析提供更多可验证的新信息。
分析性文字容易成为重复已知内容的"包装"，而新引文才是真正扩展了知识边界。

## 核心资源利用优先级

### 优先级 1：corpus_search（最高价值）

**位置**：`wiki/scripts/butler/corpus_search.py`

**使用方法**：
```bash
python3 wiki/scripts/butler/corpus_search.py "关键词" --max 15
```

**为什么是最高优先级**：
- 直接检索原文 294 卷所有段落
- 支持 PN 编号定位，精确到卷-段
- 返回结果含上下文，便于选择最合适的引文

### 优先级 2：卷节页面

**位置**：`wiki/public/pages/di/第NNN卷.md`（卷页位于 di/ 桶）

**使用场景**：读取某卷完整内容，了解人物在该卷中的全部活动。

```bash
# 查看某卷人物出现情况（卷页位于 di/ 桶）
grep -n "人物名" wiki/public/pages/di/第NNN卷.md
```

### 优先级 3：相关事件/概念页面

**位置**：`wiki/public/pages/*/` 各分桶目录中 type=事件 或 type=战役 的页面

**使用场景**：检查已有独立事件页（如 `li/赤壁之战.md`），添加双向链接。

```bash
# 递归查找事件/战役页面
grep -rl "type: 事件\|type: 战役" wiki/public/pages/ | head -20
```

## 编辑流程

### 步骤1：资源扫描（5分钟）

并行执行：
1. 读取当前页面，记录已有内容和结构
2. 用 `corpus_search.py` 搜索关键事迹的原文段落
3. 检查相关事件/战役页面是否已建
4. 确认 PN 引文分布跨卷情况

```bash
# 快速了解人物的卷目分布
python3 -c "
from wiki.scripts.page_bucket import resolve_page_file; from pathlib import Path
import re
text = resolve_page_file(Path('wiki/public/pages'), '人物名').read_text(encoding='utf-8')
for vol in sorted(set(re.findall(r'（(\d{3})-', text))):
    cnt = text.count(f'（{vol}-')
    print(f'第{int(vol)}卷: {cnt}处')
"
```

### 步骤1b：跨卷互见搜索（必须执行）

**为什么是必须步骤**：《资治通鉴》同一个人物散见多卷，人物的专属事迹只是基础——其他卷中他人对这个人的评价与描述，往往比本人的行为记载更能揭示此人的历史定位。

**执行方法**：

1. 从 `## 原文引文` 节获取卷目分布（页面已有）
2. 对每个**非主角核心卷目**，用 `corpus_search.py` 搜索该人物名

```bash
# 搜索某人物在所有卷目中的出现
python3 wiki/scripts/butler/corpus_search.py "人物名" --max 30

# 重点找他人评价
python3 wiki/scripts/butler/corpus_search.py "人物名.*曰\|人物名.*道\|人物名.*评\|人物名.*叹" --max 15
```

3. 重点寻找以下类型的跨卷材料：
   - **他人评价**：君主/同辈/对手对此人的直接评语
   - **作者史论**：司马光的"臣光曰"评议
   - **对照段落**：与同类人物的比较
   - **结局注脚**：在后文中出现的追述/盖棺定论

**典型案例**：

| 人物 | 核心卷目 | 跨卷发现 |
|---|---|---|
| 商鞅 | 第002卷变法 | 第006卷死后秦国施政仍沿商鞅之法 |
| 项羽 | 第008-011卷楚汉 | 第011卷刘邦论项羽之失，司马光评述 |
| 诸葛亮 | 第069-078卷三国 | 第082卷后人对武侯的追忆与评价 |

### 步骤2：缺口识别（重要！）

对照当前页面内容，从资源中找出**已有页面没有**的内容：
- 重要场景是否有原文引用（不只是叙述）
- 跨卷视角（其他人对此人的评价）
- 相关事件页链接
- 原文引文的卷目分布是否均衡
- 司马光史论（臣光曰）是否已收录

**最常见的缺口**：

| 缺口类型 | 频率 | 案例 |
|---|---|---|
| 重要场景有描述但无原文引文 | 高 | 有"商鞅变法"但无立木取信原文 |
| 跨卷评价缺失 | 高 | 商鞅在后代史书中的评价 |
| 相关事件页未链接 | 中 | 长平之战页在廉颇页未链接 |
| 引文集中在某1-2卷 | 中 | 人物跨30卷但引文只来自3卷 |
| 臣光曰未收录 | 中 | 司马光对该人物有专门史论 |

### 步骤3：内容写作标准

**原文引用规范**：
- 使用 `> *"原文"*（NNN-PPP）` 格式
- 每个重要事迹至少 1 处原文
- 引文后简要说明其历史意义

**PN编号引用规范**：
- 格式：`（NNN-PPP）`
- 重要事件必须有 PN
- 允许区间：`（NNN-PPP）至（NNN-PPP）`（"至"在括号外）

**段落结构**：每个重要事迹：
1. 背景/时间（朝代/年号）
2. 原文引用
3. 历史意义/影响

### 步骤4：编辑报告

每个页面编辑完成后写 `wiki/memory/reflections/edit_report_词条名.md`，包含：
1. 资源收集情况
2. 主要编辑内容表格
3. 质量评估（原页优点 + 缺口 + 未来可改进点）

## 质量标准

### Featured 精品页标准（6维度）

| 维度 | 最低要求 | 优秀标准 |
|---|---|---|
| 完整性 | 覆盖主要事迹 | 无重要事迹遗漏，含人物结局/历史影响 |
| 原文引用 | 每卷≥1处 | 关键场景均有原文，>30%内容为引文 |
| 多角度 | ≥3人他人评价 | 包含君主、同僚、对手、史家多方 |
| 结构化信息 | 基本信息节 | 相关表格（家族/官职/事迹年表等） |
| 知识图联通 | 主要人物有链接 | 相关事件页互联，官职/地名有链接 |
| 史学分析 | 有历史评价/臣光曰 | 含多角度史学评价和历史影响讨论 |

### 典型精品页结构

```markdown
# 人名
[一段话定位：身份/时代/历史地位]

## 基本信息
[朝代/官职/生卒/别号等结构化信息]

## 主要事迹
[分场景，每场景有原文引用和PN]

## 人物关系 [表格]
[与其他历史人物的关系矩阵]

## 他人评价
[多角度评价，含原文引用]

## 臣光曰
[司马光史论原文（如有）]

## 历史影响
[对后世制度/历史进程的影响]

## 相关事件
[event/战役页面链接]

## 相关人物
[wikilink列表]
```

## 执行前/中/后检查清单

**执行前**：
- [ ] 读取当前页面，了解已有内容
- [ ] 用 corpus_search 搜集关键引文
- [ ] 检查相关事件/战役页面是否存在

**执行中**：
- [ ] 每个新增场景有原文引用和PN
- [ ] 相关事件页已添加链接
- [ ] **不删改原有正确内容，只追加**
- [ ] 保留 Append-Only 原则：已有节/字段不动，只新建

**执行后**：
- [ ] 写 edit_report_词条名.md
- [ ] 调用 `record_revision.py` 留下 revision 记录
- [ ] 检查 PN 格式是否正确（NNN-PPP）
- [ ] 确认引文使用全角引号
- [ ] 运行 `/wiki` 发布（若需立即上线）
